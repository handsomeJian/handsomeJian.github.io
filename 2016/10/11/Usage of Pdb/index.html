Usage of Pdb

Start Debugging

  • Just use pdb yourfile.py to start debugging from the first line of your program.
  • Or you can type the following code in your program:

h(elp)

  • Use h to print all the available command.
  • Use h [command] like help l to print the usage of the command.

l(ist)

Show the code block that will run later.

‘b(reak)’

  • Only use b to show all the breakpoints.
  • Use b [linenum] like b 233 to set a break point at the line.
  • Use b [fuctionname] to set a break point at a function.

‘tbreak’

Use this to set a temporary breakpoint.

‘commands [bpnumber]’

Specify a list of commands for breakpoint number bpnumber. The commands themselves appear on the following lines. Type a line containing just end to terminate the commands. An example:

To remove all commands from a breakpoint, type commands and follow it immediately with end; that is, give no commands. With no bpnumber argument, commands refers to the last breakpoint set.

cl(ear)

  • Use cl to clear all the breakpoints.
  • Use cl [num] clear one breakpoint which number is num.

disable/enable [num]

Disable/Enalbe a certain breakpoint.

n(ext)

Make the program run to next line but won’t get into a function.

s(tep)

Like next but will get into a function.

c(ont(inue))

Let program run to the next breakpoint.

j(ump) [linenum]

Jump to the given line.

p [argname]

Print the argument.

!

Use ! following with code can change the value of an object. For example:

q(uit)

Quit pdb.

Tips:

  • Directly press enter to repeat the former command.
